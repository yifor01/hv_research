# 主管摘要

## 本名單用途

本文件列出**未入主要 Top 15 名單**的所有備選候選 PI，作為：

1. 主要 Top 15 任一位婉拒合作時，快速調閱替代候選
2. 主管被問「為什麼沒選 XX」時的 FAQ 應答池
3. HR / 內部獵才團隊做跨題目人才地圖

本名單與主報告（Top 15）**不重複**，意即名單內任何一位在主報告中都不會以候選身分出現。若主名單中某位 PI 被比較為「技術替代」，此處才會以對照方式出現其名字。

## 使用建議

1. **優先順序**：A → B → C。**不要跳過 A 類直接用 C 類**（跨域成本高）。
2. **互補配對**：若主名單某位因綁定被排除，優先看「可取代主名單誰」欄位映射，而非按分數逐次下替。
3. **時效性**：E 類失效案例（水野潤、彭文志、張耀文）告訴我們：**企業內部借調 / 任命動態公開 web 不可見**。實際接觸前仍須透過業界管道 double-check 現職狀態。
4. **文件形態**：本名單目標 20-30 頁，比主報告短。**細節深度不如主報告**，設計為「掃過即可決定是否納入追蹤」。

## B. 評估方法論簡述

5 維度評分框架（每維 0-10 分，總分 10）：

1. **技術契合度** — 與 TSMC 7 大技術題目（T1 APC / T2 EDA / T3 良率 / T4 封裝 / T5 Device / T6 Digital Twin / T7 Vision+RAG）的對齊程度
2. **產業落地性** — 開源證據、GitHub、產業顧問、TRL 成熟度
3. **學生工程素養** — Lab 規模、畢業生流向、AICUP / Kaggle / 競賽紀錄
4. **合作開放性** — 是否有企業獨占綁定、冠名講座、JDP / ARP 狀態、COI 透明度
5. **長期可持續** — 年齡、行政負擔、研究動能（近 5 年頂刊 / 主會論文密度）
