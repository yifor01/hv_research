# PI 長期投資盡職調查框架（5 維度 × 0-2 分，可重用模板）

- **版本**：v1.0（2026-04-22 首發）
- **驗證案例**：`reports/2026-04-tw-univ-semi-ai-professors/`（TSMC AI 部門 × 43 位學界 PI，Phase 1-3 共 3 階段）
- **適用場景**：任何企業想找學界 PI 做 **5-10 年長期合作**（人才共建 + 技術共研），不是單次委託案/顧問合約
- **不適用**：
  - 單次 POC 評估（過於大砲打小鳥）
  - 學界內部升等/評鑑（框架偏重企業視角）
  - 短期 1-2 年技術採購（維度 5 黃金期剩餘不成立問題）

---

## §0 為什麼是 5 維度？

從 20 年「企業-學界合作實務失敗案例」抽出 5 大失敗原因反推，每個維度各對應一類失敗：

| 維度 | 對應失敗模式 | 踩過的坑 |
|---|---|---|
| 1. 技術命中度 | PI 方向看似相關，實際無具體 deliverable | "方法論可遷移" ≠ "PI 真的做過這 domain" |
| 2. 學生招募潛力 | 5 年內只拿到 2-3 位畢業生，人才流入斷炊 | 小 Lab 或 solo PI 的長期風險 |
| 3. 長期 Lab 開放度 | 合作 3 年後 PI 重心轉移，合作變鬆散 | 無企業共建 Lab 先例 = 關係難深化 |
| 4. 資源未被搶佔 | PI 被競爭廠商 Director 級綁定，投入度打折 | 王鈺強反例（NVIDIA Director）= TSMC 變第二順位 |
| 5. 個人黃金期 | PI 升任行政職/借調/CEO 轉任，5 年內離開第一線 | 副院長 + 系主任 + 借調 = 實際研究時間 < 20% |

## §1 5 維度評分錨點

每維度評分 **0 / 0.5 / 1 / 1.5 / 2**；各錨點具體描述如下。

### 維度 1：技術命中度（0-2 分）

> **核心問題**：PI 的核心技術方向，與你想做的題目，**有多直接命中**？

| 分數 | 錨點 |
|---|---|
| **2.0** | PI 近 3 年有 ≥2 篇論文/專利直接做這個題目（domain 100% 吻合），且方法論可用於 production 級驗證 |
| **1.5-1.8** | PI 近 3 年有 1 篇直接命中 + ≥3 篇相近題目，或方法論完全吻合但 domain 移植需要 < 6 個月 |
| **1.0-1.3** | PI 方法論可遷移（上位概念相同），但需 6-12 月 domain onboarding |
| **0.5** | 上位類別相關（例如都做「ML」），但缺少具體接近的研究 |
| **0.0** | 完全無 domain 相關論文 / 專利 / 計畫 |

**判斷技巧**：
- 查 Google Scholar 近 3 年論文，搜尋具體 domain 關鍵詞（如「Cu-Cu bonding」「RAG」「EUV hotspot」）
- 至少找 2-3 篇論文比對 abstract 與你的題目——**不要看「研究方向自述」，要看實際論文**
- 方法論相似 ≠ 命中。Speech 情感識別方法論可「遷移」到製程異常，但這是 1.0 不是 2.0

### 維度 2：5 年學生招募潛力（0-2 分）

> **核心問題**：5 年內 PI 能幫你帶來多少新員工？質量如何？

| 分數 | 錨點 |
|---|---|
| **2.0** | Lab 25+ 人（系所最大）+ 學生每年 ≥3 位直接進你公司 + 頂會一作（IEEE/ACM top tier） |
| **1.5-1.8** | Lab 12-20 人 + 每年 1-2 位進你公司 + 頂會一作或頂刊一作 |
| **1.0-1.3** | Lab 8-12 人 + 學生流向你公司的同領域（競爭廠商/整合業界） + 中等會議品質 |
| **0.5** | Lab 5 人以下 / solo PI / 學生 100% 流向海外或學界 |
| **0.0** | 無研究生（純教學教授）/ Lab 重組中 / PI 即將退休 |

**判斷技巧**：
- 先查官方 Lab page（人員列表）
- 交叉查 LinkedIn 畢業生去向（最近 3 年）
- 頂會一作 = 學生能主導研究（非掛名），這是工程素養指標

### 維度 3：企業共建長期 Lab 開放度（0-2 分）

> **核心問題**：PI 是否願意接受 5 年以上的深度合作？有無先例？

| 分數 | 錨點 |
|---|---|
| **2.0** | 已有企業冠名 Lab / JDP / 長期顧問合約 + 正在活躍中 + 制度框架完整（IP / NDA 都有範本） |
| **1.5-1.8** | 曾主持 ≥2 個產學計畫（NSTC 產學聯盟等）+ 有企業委託研究論文發表紀錄 |
| **1.0-1.3** | 有零星產學合作但多為短期（1 年以內）+ 願意試長期但無先例 |
| **0.5** | 以 NSTC 一般型計畫為主，近 5 年無企業合作紀錄，但研究主題企業能接 |
| **0.0** | 純學術路線（Nature/Science/ACL）+ 無任何企業合作紀錄 |

**判斷技巧**：
- 搜尋「PI 姓名 + 產學合作」「PI 姓名 + JDP」「PI 姓名 + 冠名教授」
- 查論文 Acknowledgments 的資金來源（排除 NSTC 基礎型，看企業贊助）
- 「台積電-X 大聯合研究中心」成員 ≠ 個人已有合作，要再確認該 PI 個人是否在名單內

### 維度 4：資源未被搶佔程度（0-2 分，**最關鍵**）

> **核心問題**：如果你現在投入，PI 會優先給你，還是排在某某大廠之後？

| 分數 | 錨點 |
|---|---|
| **2.0** | **完全自由身**：無任何大廠 Director 級職務 / 冠名 Chair / JDP 主持人 |
| **1.5-1.8** | 弱綁定（一次性學生獎、單篇合著、校友身份 10 年前）/ 或「與你同陣營」（見下方同陣營判準）|
| **1.0-1.3** | 中度綁定（曾任某大廠 Chair Professor，但已 3 年前結束）/ 或多家大廠分散合作（= 平衡） |
| **0.5** | 深度綁定（現任大廠 Chair Professor + 獨占 IP）/ 顧問級但時間占比 10-20% |
| **0.0** | **王鈺強反例級**：大廠 Director / Deputy Director / Joint Center 副主任以上，時間占比 30%+ |

**大廠同陣營判準**（不扣分）：
- 國家級共投機構（如台灣 SAT 半導體學院，NSTC + TSMC + 多家共投）= 同陣營
- 你公司本身（TSMC 投資找 TSMC 合聘教授）= 中性不加分（不因既有合作加分，因為新合作目的是 ADD value）
- 你的上下游非競爭方（TSMC 投資找 Intel 顧問 PI = 深度綁定扣；TSMC 投資找 Intel 材料供應商 PI = 不扣）

**判斷技巧**：
- 必做 Web search：「PI + NVIDIA」「PI + Google」「PI + Meta」「PI + 主要競爭廠商」
- 查官方 bio 上是否列出「XX Chair Professorship」
- 查 Joint Center 官網是否列名為 Deputy / Director

### 維度 5：個人黃金期剩餘（0-2 分）

> **核心問題**：5-10 年內 PI 是否仍是活躍第一線研究者？

| 分數 | 錨點 |
|---|---|
| **2.0** | 40-50 歲 + 無行政職 + 近 5 年連續高產 + 無升任院長級別跡象 |
| **1.5-1.8** | 45-55 歲 + 系主任/所長級輕行政（20% 以內）+ 論文產出穩定 |
| **1.0-1.3** | 55-65 歲 + 中等行政（副院長 / 系主任，20-40% 時間）+ 仍有主導論文 |
| **0.5** | 60+ 歲即將退休 / 重度行政（院長 + 副校長，40-60%）/ 借調政府單位 |
| **0.0** | 已宣告退休 / 轉任 CEO 級 / 長期借調（> 2 年） |

**判斷技巧**：
- 看最近 3 年論文產量趨勢（遞增/穩定/下降）
- 查行政職務清單（校內 + 校外）
- 若是副校長 / 研發長，通常 3-4 年任期，結束後會釋放能量

## §2 綁定折扣判斷樹（維度 4 細節）

```
PI 有外部企業/大廠關係？
├── 無 → 2.0
├── 有 → 進一步判斷
    ├── 是你公司本身 → 中性 1.5（新合作不因既有合作額外加分）
    ├── 是你的同陣營（國家級共投機構、你的上下游非競爭方）→ 1.5-1.8
    ├── 是你的競爭對手
    │   ├── Director / Deputy Director / Joint Center 副主任以上 → 0.0（王鈺強級）
    │   ├── Chair Professor 現任 → 0.5
    │   ├── Chair Professor 已結束（3+ 年前） → 1.0-1.3
    │   ├── 顧問級（時間 < 10%）→ 1.3-1.5
    │   └── 一次性合著 / 學生獎 → 1.8
    └── 是中性第三方（非競爭非同陣營）→ 1.5-1.8
```

## §3 標準工作流程（Phase 1 → 2 → 3）

### Phase 1：Tier 分類 + 隱形綁定初篩（每位 PI 約 15 分鐘）

**產出**：`phase1-candidates.md`（一行一位 PI，含下列欄位）

| 欄位 | 內容 | 判斷方式 |
|---|---|---|
| PI | 姓名 / 英文名 / 校系 | 官方頁面 |
| Tier | 主力 / 次選 / 觀察 | 快速判斷題目契合 |
| 領域 | 主研究方向（3 個關鍵詞以內）| Google Scholar 最近 10 篇論文 |
| 近 3 年頂會 | 是 / 否（題目相關頂會）| DBLP + Google Scholar |
| 隱形綁定 | 🟢 無 / 🟡 弱 / 🔴 深 | Web search + 官方 bio |
| 推進 Phase 2？| Y / N | 綜合上面 4 欄 |

**硬門檻**（未達則不進 Phase 2）：
1. 近 3 年 ≥1 篇相關頂會 / 頂刊
2. Lab ≥3 人
3. 隱形綁定非 🔴

### Phase 2：深度 profile（每位 PI 約 3-4 小時，可平行化）

**產出**：`phase2-profile-{slug}.md` 每位一份

**章節**：
1. 基本身份 + Lab 資訊
2. 隱形綁定詳查（近 3 年論文 Acknowledgments / 合約公開紀錄 / 個人 bio）
3. 技術契合度（代表論文 5 篇 + 對你題目的遷移性分析）
4. 學生工程素養（Lab 規模 / 競賽 / 畢業生去向）
5. 合作優缺點 + 3 個具體合作題目建議
6. 資料來源清單（URL + 訪問日期）

**工具**：WebSearch + WebFetch（Google Scholar / DBLP / 官網 / NSTC 獎項頁）；可用 Claude Agent 平行化產出

### Phase 3：企業視角 5 維度重評（每位 PI 約 30 分鐘）

**關鍵步驟**：用你**自己公司**視角（而非「競爭對手」通用視角）重新評分

- 維度 4 重新定義「誰是同陣營」vs「誰是競爭對手」
- 可能產生大幅排名翻轉（例如 TSMC 視角下 SAT 合聘從綁定 → 同陣營，PI 從 #6 → #1）

**產出**：`phase3-tsmc-view-reeval.md`（參考 `phase2-tsmc-reeval-5pis.md` 格式）

**計算公式**：
```
分數 = 維1 + 維2 + 維3 + 維4 + 維5
範圍：0-10
```

### Phase 4（選配）：候選池擴大

- 若 Top N 在某題目只有 1 位 PI，主管會質疑「為何單兵作戰」
- 啟動 Batch agent 找 2-3 位補強（參考 `phase3-batch-a-llm-rag-candidates.md` 格式）
- 目標：每個關鍵題目至少 3 位 PI 互補

## §4 換公司視角時的維度調整

如果你要幫**非 TSMC 的公司**用這個框架（例如 MediaTek / Intel / NVIDIA 的學界 PI 盤點）：

### 需重新定義的欄位

| 欄位 | TSMC 視角 | 其他公司視角（以 MediaTek 為例）|
|---|---|---|
| 「同陣營」機構 | SAT 國家半導體學院 / NSTC 共投 Lab | MediaTek 研究院 / IC 設計共投 / 聯發科慈善 |
| 「競爭對手」 | Samsung Foundry / Intel Foundry / SMIC | Qualcomm / Apple / Broadcom / UNISOC |
| 「中性 / 上下游」 | 設備商（KLA / AMAT / ASML）/ 材料商 | 晶圓代工廠（TSMC/UMC）/ OSAT |

### 維度 1-3 通常不需調整，但需注意題目換了之後：
- 「2nm 命中」變成「MediaTek 7nm APU + 能耗」之類
- 先定義你的 3 個核心投資目的（例如 TSMC 是「量產速度 + 良率 + 人員工作效率」）再評分

## §5 常見判斷陷阱

1. **「方法論可遷移」≠ 命中 2.0**
   - Speech/Affective Computing → 製程異常檢測，方法論相似但 domain 完全不同，應打 1.0 或更低，不能因為 PI 論文多就給高分
2. **「TSMC 合作紀錄」≠ TSMC 視角加分**
   - 新合作的目的是 ADD value，既有合作只是中性；如果把 TSMC 合作當加分 = TSMC 只會重複投資同一 PI，無法擴大生態
3. **行政職重 ≠ 立即排除**
   - 副院長/系主任 3-4 年輪替，等卸任前 1 年接觸，可能是最佳時機
4. **Lab 大 ≠ 學生品質高**
   - 25 人 Lab 如果近 3 年沒有一作頂會，表示 PI 帶人能量可能過載
5. **「無 TSMC 合作」≠ 綁定風險低**
   - 要查是否有其他大廠 Director 級（王鈺強反例）；無 TSMC 合作 + 有 NVIDIA Director = 綁定最深
6. **頂會一作看主角不看掛名**
   - IEEE Fellow 是個人榮譽，不代表 Lab 學生發得出；要看學生 first-author 的頂會論文比例

## §6 輸出物清單（standard deliverables）

完整走一輪 Phase 1-3（可選 Phase 4）後應產出：

```
reports/YYYY-MM-<topic>/
├── phase1-candidates.md          # Tier 分類初篩
├── phase2-profile-*.md           # 每位 PI 深度 profile
├── phase2-tier1-verification.md  # Tier 1 候選人交叉驗證
├── phase2-final-ranking.md       # 綜合排序（通用視角）
├── phase3-<company>-view-reeval.md  # 企業視角 5 維度重評
├── phase3-batch-<topic>-candidates.md  # 補強題目候選（選配）
├── phase3-integrated-ranking.md  # 最終 Top N 整合
└── <company>_Top<N>_長期投資分析.pdf  # 主管決策文件
```

**PDF 產出**：`scripts/md_to_pdf.py`（本專案附），支援多 MD 合併、封面、目錄、emoji → 樣式化標記替換。

## §7 案例參考

本框架的首個驗證案例：TSMC AI 部門 × 43 位台灣學界半導體 PI（2026-04）

- **Phase 1**：60+ 位 → 篩出 21 位主候選 + 16 位 AI 名人未入選
- **Phase 2**：21 位深度 profile + 通用視角 v2 排序
- **Phase 3**：TSMC 視角 5 維度重評 → Top 7 + ⚠️ 5 位重評
- **Phase 4**：補強題目候選（Batch A 補 3 位 LLM/RAG + Batch B 補 3 位封裝）→ Top 15 最終版

**關鍵學習**：Phase 3 的視角翻轉造成 PI 排名大幅變動（馬誠佑從 #6 → #1，王俊明從 #13 → #1），驗證「通用視角 vs 企業視角」的差異絕非邊際，而是**決策級別**。任何不做 Phase 3 的 PI 盤點都是半成品。
